    pushTarget暂停依赖收集原因分析

    一路败北 10/28/2021 vue

    pushTarget 调用时没有传 target 对象，其实也就是没有传 watcher，导致 Dep.target 为 undefined，所以此时依赖收集会暂停。pushTarget()主要是在 data 初始化的 getData 以及 callHook 里面调用了，详细见https://github.com/vuejs/vue/commit/318f29fcdf3372ff57a09be6d1dc595d14c92e70 (opens new window)，下面分析一下原因

    export function pushTarget(target: ?Watcher) {
  targetStack.push(target)
  Dep.target = target
}

    在这里可以看到，在调用 data()时，暂停了依赖收集。源码中说明了，是为了解决https://github.com/vuejs/vue/issues/7573 (opens new window) 这个 bug

    function initData(vm: Component) {
  let data = vm.$options.data
  data = vm._data = typeof data === 'function' ? getData(data, vm) : data || {}
  //...
}

export function getData(data: Function, vm: Component): any {
  // #7573 disable dep collection when invoking data getters
  pushTarget()
  try {
    return data.call(vm, vm)
  } catch (e) {
    handleError(e, vm, `data()`)
    return {}
  } finally {
    popTarget()
  }
}

    这个 bug 的现象是当子组件 data 写成函数形式并且函数中使用了父组件传给子组件的 props，当父组件中做为 props 传入子组件的那个响应式数据改变时，会触发两次父组件的更新

    之所以会这样，是因为执行 data.call(vm, vm) 获取子组件 data 值时，里面使用了 props，此时会触发 props 的 getter，造成 props 收集依赖。由于数据初始化的时机是 beforeCreated 和 created 两个生命周期之间，此时还没有进入子组件的渲染阶段（生成子组件的渲染 Watcher 是在 mountComponent 中），也就没有子组件的渲染 Watcher。

    所以这时候 Dep.target 指向的依然是父组件的渲染 Watcher，于是 props 收集了父组件的 watcher，导致父组件更新两次，一次是父组件自身 data 中的收集到的渲染 watcher 派发的更新，另外一次是子组件 props 也是响应式的，子组件 props 里面收集到的父组件的渲染 watcher 派发的更新。

    从 GitHub 的修复记录来看，在 callHook 中也暂停了依赖收集，原因同上，子组件的 beforeCreate、created、beforeMount 这三个生命周期钩子函数执行的时候，Dep.Target 指向的还是父组件的渲染 watcher。
